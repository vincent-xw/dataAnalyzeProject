# 模板 Prompt 预设与放大编辑设计

## 目标

降低首次创建分析模板的成本：预填符合当前受控脚本和报表协议的加工、报表 Prompt，并提供大面积编辑空间。

## 交互

模板创建页的两个 Prompt 字段均使用同一套编辑器。页面内展示可读摘要和“放大编辑”按钮；点击后打开模态窗口，在更大的 `textarea` 中查看和编辑。用户点击“保存”才同步内容，点击“取消”保留打开前的内容。

该交互只覆盖新建模板页，不改变分析请求页和报表创建页的运行时 Prompt 编辑方式。

## 默认内容

加工 Prompt 要求模型只在启用的精确脚本版本中决策，依据标准字段、脚本元数据和用户需求给出结构化结果；禁止生成代码、读取原始行或猜测不存在字段。

报表 Prompt 要求模型仅生成 `ReportConfigSchema` 支持的固定组件和筛选器，使用脚本结果 Schema 中的字段及中文标题；禁止 HTML、CSS、JavaScript、表达式和虚构字段。

## 数据与兼容性

默认值只在新建页面的初始状态生效，仍通过现有 `POST /api/templates` 保存。用户在弹窗中保存的文本会成为模板的 v1 Prompt；后续运行时编辑仍通过现有 Prompt 版本接口生成新版本。

## 验收与测试

- 新建模板页首次渲染时展示两份预设 Prompt。
- 点击任一“放大编辑”按钮可打开对应标题和完整文本。
- 保存会更新页面字段；取消不会更新。
- 创建请求携带用户最终保存的两份 Prompt。
