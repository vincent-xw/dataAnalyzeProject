# 基础数据结果与审核型客制化脚本设计

## 目标

消除“LLM 再生成中文字段说明”造成的映射歧义；字段映射完成后自动生成可报表化的基础数据；复杂计算仅以用户审核后创建的 GitHub 候选脚本进入生产执行链路。

## 字段模型与映射归属

模板字段使用 `sourceLabel`、`name`、`type`、`required`：`sourceLabel` 必须是生成模板时数据表中的原始表头，`name` 是稳定英文 snake_case 内部键。LLM 必须逐项返回输入表头中的 `sourceLabel`，Worker 校验其与检查结果一一对应；不允许模型生成、翻译或改写中文展示名。

`field_mappings` 改为归属 `dataset_version_id`，而非模板。一个数据集版本保存“实际来源表头 → 模板英文键”的映射，页面与报表显示当前版本的实际来源表头，例如 `销售额 → sales_amount`。模板从真实表头生成时可按 `sourceLabel` 自动预选；没有唯一匹配时维持未映射。

既有模板和映射不进行静默转换：旧 `description` 不能被当作真实表头。已有数据需重新从数据表生成模板字段或重新确认映射，避免继承历史中文不一致。

## 默认基础数据结果

字段映射确认成功后，Worker 创建不依赖 LLM 的 `baseline` 任务并投递 Queue。任务只读取所选工作表/CSV，按当前数据集版本映射转换字段名和类型，以 NDJSON 写入 R2；同时写入结果 Schema 和行数摘要。它不进行求和、求差、聚合或推断。

基础任务完成后可直接创建报表。报表读取英文内部键的数据，但用该数据集版本映射中的来源中文表头作为组件标题、筛选标签和表格列名。基础结果保留为后续客制化任务的标准化输入。

## 加工页与客制化脚本

加工页的 Prompt 之前展示当前数据集版本已映射字段清单，格式为 `来源表头 → 英文键（类型）`。这份清单来自版本映射与模板字段，不暴露原始数据行。

用户填写客制化需求后，LLM 仅接收字段清单、受限脚本 SDK 协议和需求，生成候选 TypeScript、脚本元数据、参数 Schema 与说明。页面展示完整候选代码和静态校验结果；只有用户明确点击“创建候选 PR”时，才调用既有 GitHub 候选脚本接口。

候选 PR 继续沿用现有 CI、人工合并、Worker 部署和脚本目录同步链路。未合并的候选脚本不能执行；部署后用户重新发起加工，由现有脚本推荐与确认流程选择已启用版本。

## 安全与验证

LLM 不接收原始数据行、R2 地址或密钥。默认任务和客制化脚本均流式读写，输出 Schema 在写入最终结果前校验。测试覆盖字段原样保存、版本级映射隔离、基础任务结果、加工页字段清单、候选代码预览与“未确认不创建 PR”。
