# 无模板数据资产导入设计

## 目标

让用户从“我的数据”直接上传 CSV 或 XLSX，在不选择模板、不执行字段映射的前提下将一个表转换为 R2 中的 NDJSON 数据资产；资产随后可维护元数据、预览，并作为图表渲染和后续加工的直接输入。

## 用户流程

“我的数据”页提供上传入口。CSV 用户选择编码与分隔符；XLSX 上传后读取工作表目录，用户必须选择一个工作表。确认后服务端解析该单表，写入原始文件、NDJSON 数据、字段 Schema 和预览对象，创建资产记录并跳转资产详情页。名称默认使用文件名，描述与标签为空，可在详情页维护。

## 数据与 API

新增独立的资产上传接口，不复用要求 `x-template-id` 的数据集上传接口。接口返回等待选择工作表或已创建资产；完成时保存 `data_object_key`、`schema_object_key`、`preview_object_key`、行数和文件来源信息。NDJSON 字段值保持源数据语义，不强制套入模板类型；Schema 仅描述推断出的列名与展示类型。

数据资产不再保存 `template_id`。资产列表、详情、元数据更新及元数据建议不再读取模板；列表只显示资产自身元数据。

本项目目前仅包含测试数据，因此采用破坏性迁移：删除旧模板、Prompt、数据集、数据集版本、字段映射、执行计划、任务、报表与旧资产记录；同时清理 R2 中旧 `data-analyze/datasets/`、`data-analyze/tasks/`、`data-analyze/reports/` 和 `data-analyze/assets/` 前缀。迁移后只保留无模板资产导入链路；旧数据集、映射和任务 API 与前端入口一并删除，而非兼容保留。

## R2 结构与限制

每个上传资产使用 `data-analyze/assets/<assetId>/` 前缀，包含 `source/original.<ext>`、`data/data.ndjson`、`schema.json` 与 `preview.json`。沿用现有 10 MB 上传限制、CSV 编码和分隔符规则，以及 XLSX 单工作表选择。解析或写入失败时清理本次创建的对象和控制面记录。

## 后续集成

图表渲染入口直接选择 `data_assets` 并使用其 Schema 与 NDJSON。模板、字段映射、脚本加工、分析计划、任务与报表等旧链路不属于本次新模型，随测试历史数据一起移除。

## 验证

Worker 测试覆盖 CSV 导入、XLSX 等待/选择工作表、无模板资产列表与预览、失败回滚；前端测试覆盖从资产页发起上传、工作表选择与完成跳转。运行范围测试、类型检查、构建，并记录现有无关失败。
