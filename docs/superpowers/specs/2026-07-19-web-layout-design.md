# 前端整体布局（第一轮）设计

## 目标

将 Web 应用的页面外壳改为 Ant Design 的标准上中下布局，提升整体导航与内容区的一致性。顶部保留既有导航，业务页面、后端逻辑、接口路径及请求字段均不在本次范围内。

## 方案

采用 `Layout > Header + Content + Footer`：

- `Header` 高度为 64px，使用深色主题色，包含左侧白色产品名“NEXUS 数据智能平台”和右侧横向主菜单。
- 主菜单使用 Ant Design 深色 `Menu`，沿用现有路由与文案：`/assets`、`/analyses`、`/settings`、`/assets/upload`；活动路由需以高对比度状态自动高亮。
- `Content` 使用浅灰页面背景与最小视口高度。其内部新增居中工作区容器，最大宽度约 1200px，并保留适合当前业务页面的纵向留白。
- `Footer` 采用低干扰的版权文本“© {当前年份} NEXUS 数据智能平台 · Created by vincent-xw · All Rights Reserved.”，其中年份在页面运行时自动生成；内容不足一屏时页脚仍位于页面底部。
- 窄屏下，Header 内部可横向滚动，品牌与菜单不换行、不重叠；内容区左右间距收窄。

## 组件边界与数据流

- 只修改 `apps/web/src/App.tsx`，使其负责应用壳层、导航与 `Outlet` 的承载。
- 只修改 `apps/web/src/styles.css` 中与应用壳层相关的全局样式，保留现有业务页面类名的语义。
- 不修改 `apps/web/src/router.tsx` 的路由声明，不修改任何 `features/` 页面、API 客户端、Worker 或共享协议。
- 菜单导航通过既有 React Router 路由跳转，不新增请求、状态或字段映射。

## 验收与测试

- 不为本轮前端视觉文案调整新增自动化测试；通过本地预览确认标题、动态年份和版权文本展示正确。
- 运行前端 typecheck 与前端构建，确认 Ant Design 布局与现有页面可兼容。
- 人工检查桌面端与窄屏导航：当前菜单项高亮、内容区居中、页面内容不足一屏时页脚贴底。

## 非目标

- 不引入侧栏、账户菜单、通知、主题切换或页面级 Ant Design 组件迁移。
- 不调整业务文案、路由结构、后端接口、请求字段与返回值处理。

## 第二轮：Ant Design 数据与表单组件

本轮将 Web 页面中所有原生表格、按钮、输入框、选择框、复选框和文本域替换为 Ant Design 组件：

- `assets` 列表与预览、分析页预览及报告表格使用 `Table`；保持现有列、数据内容、横向滚动和无分页展示。
- 所有业务动作使用 `Button`；上传入口通过既有前端路由跳转，不改变路由地址。
- 元数据、上传、分析和系统设置表单使用 `Form`、`Input`、`Input.TextArea`、`Select`、`Checkbox` 与 `Upload`；文件选择仅收集文件，不自动上传，原有提交函数、状态和 API 请求保持不变。
- 不新增自动化测试；通过逐页本地预览验证控件状态、数据展示和原有交互。

## 第三轮：资产列表标签与面板细节

- assets 页面中的 `panel` 不显示边框，保留白色背景、圆角和轻阴影。
- 资产标签使用 Ant Design `Tag` 预设色；颜色由标签文本的确定性哈希映射到固定调色板，同一标签在所有刷新和数据资产中保持相同颜色。
- 不修改标签数组、API 请求或空标签状态；不新增自动化测试，通过本地预览验证。
