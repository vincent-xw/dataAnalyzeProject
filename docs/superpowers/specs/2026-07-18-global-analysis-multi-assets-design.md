# 全局多数据表分析设计

数据分析成为顶层入口，不再从单一资产详情进入。`analyses` 保存需求、规则、状态和时间；`analysis_data_assets` 以 `primary`/`reference` 关联一张主表和零至多张参考表。新建页先多选资产并指定主表，再提交自然语言需求。模型和首版图表只读取主表；参考表仅被保存和展示，为后续跨表规则预留。

迁移把现有 `analyses.asset_id` 回填为主表关联，再移除该列。全局列表显示所有分析及其主表；详情显示关联表，使用主表规则和数据渲染。旧的资产内分析入口与路由移除。

校验：至少一张资产、主表属于已选资产、每个分析只有一张主表、同一资产不重复关联。保留 Worker API 与迁移测试，省略重复的细粒度组件测试。
